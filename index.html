Programming fundamentals

Basic program structure

RobotC programs are basically a sequence of statements that the computer (NXT) will run. Whitespace doesn't matter.

The next statement doesn't start until the previous statement finishes, unless you explicitly say otherwise.

There are two types of statements: command statements, and flow control statements

Command statements

Command statements are basically actions. They do something, like initializing a variable or moving a motor.

Command statements are terminated by ;s.

Control flow statements

Control flow statements control the flow of the program (semi-obviously).

For example, the if statement controls whether or not a collection of statements run, based on a condition.

Control flow statements look like this: statement (parameter) { action; action; action; }

Let's just look at an example.

// declare (initialize) a variable
bool continue = true;
 
while (continue)
{
 /*
  this is a contrived example. usually you would
  just put `someVariable <= 5` in the while loop.
 */
 if (someVariable <= 5)
 {
  continue = false;
 }
}

Common control flow statements

if (condition) {}

while (condition) {}

for (initialCommand; condition; endCommand) {}

Ancillary concepts

Application code

This is a fancy term for situation-specific code.

Application code is code that contains the logic specific to whatever problem you're trying to solve.

For example, a robot's teliop, or a particular autonomous routine.

Libraries

A library is a unit of code that's designed to be reused.

Libraries cannot be compiled on their own, instead, application programs use them

Libraries are good because they reduce code duplication, which means less work for everyone

APIs

An API, or Application Programming Interface, is a collection of command statements that works with a certain topic

For example: "the motors API"

If you want to seriously program with RobotC you will want to use the APIs that we have written

If you just want to move a prototype's motors, you'll want to use the RobotC APIs. See motors and servos on the RobotC wiki
